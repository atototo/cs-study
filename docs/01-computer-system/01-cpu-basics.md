# 🧠 CPU 기초

> CPU = 컴퓨터의 두뇌. 명령어를 읽고 실행하는 장치

## 비유: CPU는 요리사

| 요리사 | CPU |
|--------|-----|
| 레시피를 읽는다 | 명령어를 읽는다 (Fetch) |
| 레시피를 이해한다 | 명령어를 해석한다 (Decode) |
| 요리를 만든다 | 명령어를 실행한다 (Execute) |
| 완성된 요리를 접시에 담는다 | 결과를 저장한다 (Store) |

이 4단계가 계속 반복되는 게 CPU의 일이에요.

---

## 실무 연결

```javascript
let a = 1;      // CPU: "a라는 곳에 1을 저장해"
let b = 2;      // CPU: "b라는 곳에 2를 저장해"
let c = a + b;  // CPU: "a와 b를 더해서 c에 저장해"
```

이 3줄을 실행할 때 CPU는 각 줄을 **Fetch → Decode → Execute → Store** 반복하며 처리해요.

---

## 핵심 개념 3개

| 개념 | 설명 | 실무에서 들어봤을 표현 |
|------|------|----------------------|
| **코어(Core)** | 실제로 일하는 요리사 수. 4코어 = 요리사 4명 | "멀티코어", "쿼드코어" |
| **클럭(Clock)** | 요리사의 손 속도. 3.0GHz = 초당 30억 번 동작 | "클럭 속도" |
| **캐시(Cache)** | 요리사 바로 옆 작은 재료대. 자주 쓰는 것 보관 | "캐시 미스", "L1/L2 캐시" |

---

## CPU vs 코어 정리

> **CPU = 주방 (두뇌 전체)**  
> **코어 = 주방 안의 요리사 (실제로 일하는 단위)**

- **4코어 CPU** = 요리사 4명이 있는 주방 1개
- **8코어 CPU** = 요리사 8명이 있는 주방 1개

---

## 왜 코어가 여러 개일까?

```
[싱글코어 시절]
요리사 1명이 모든 주문을 순서대로 처리
→ 파스타 만들고 → 스테이크 만들고 → 샐러드 만들고...

[멀티코어 현재]
요리사 4명이 동시에 처리
→ 요리사A: 파스타
→ 요리사B: 스테이크  (동시에!)
→ 요리사C: 샐러드
→ 요리사D: 디저트
```

---

## 실무 연결 - 병렬 빌드

```bash
# 빌드할 때
npm run build  # 싱글코어로 하면 느림

# 병렬 빌드
npm run build --parallel  # 여러 코어가 나눠서 처리 → 빠름
```

---

## 🎯 면접 예상 질문

> **Q: 멀티코어 CPU에서 병렬처리가 어떻게 가능한가요?**

**A:** 멀티코어 CPU는 여러 개의 코어가 독립적으로 명령어를 실행할 수 있습니다. 각 코어가 별도의 쓰레드나 프로세스를 담당하여 동시에 작업을 처리하므로 병렬처리가 가능합니다.
